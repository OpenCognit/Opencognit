## What is OpenCognit?

OpenCognit is an **AI agent orchestration OS** вҖ” not a chatbot, not a single-agent wrapper. It runs a **virtual company** of autonomous AI agents that work together without you watching.

You set a goal. The CEO agent breaks it down, assigns tasks to specialists, reviews their work with a built-in Critic loop, and escalates blockers вҖ” while you sleep.

```
You вҶ’ Goal вҶ’ CEO Agent вҶ’ Dev Agent вҶ’ Writer Agent вҶ’ Researcher Agent
                 вҶ‘               вҶ“              вҶ“
          Persistent Memory вҶҗв”Җв”Җ Critic вҶҗв”Җв”Җ Results вҶҗв”Җв”Җв”Җв”Җв”ҳ
```

## Is OpenCognit right for you?

**Yes, if:**
- You want AI agents that actually *remember* вҖ” not just across a conversation, but across days and weeks
- Your CEO agent should reason about the company state before delegating, not just execute prompts
- You need a Critic loop вҖ” no silent "done" without quality review
- You want hard budget limits enforced at the cent level, not just estimates after the fact
- You need an org chart, not a pipeline вҖ” agents with roles, hierarchies, and peer meetings
- You're building something real and want the full control plane: Goals, Kanban, War Room, Activity Feed

**Probably not, if:**
- You just want to chain a few prompts together вҖ” use LangChain or a simple script
- You need a no-code drag-and-drop workflow builder вҖ” try n8n
- You only ever run one agent at a time

## Core Features

### рҹ§  CEO Orchestrator with Extended Thinking
The CEO agent uses `claude-sonnet-4-6` with Anthropic's **Extended Thinking** вҖ” it reasons step-by-step before making decisions. It delegates tasks via native tool calls (not JSON parsing), calls team meetings, and requests board approval for hiring.

### рҹ’ҫ Persistent Memory System
Each agent has its own memory with **Rooms** (key-value), **Diary** (structured entries), and a **Knowledge Graph** (subject-predicate-object triplets). Agents tag important outputs with `[REMEMBER:room]` вҖ” parsed and saved automatically. Old memories consolidate into LLM-compressed summaries.

### рҹ”Ғ Planner вҶ’ Executor вҶ’ Critic Loop
Every task result goes through a lightweight evaluator (Haiku) before being marked `done`. If insufficient, the agent retries вҖ” up to 2 times вҖ” before escalating to the board. No silent failures.

### рҹ”— Task Chaining
Define blocking relationships between tasks. When Task A completes, dependent tasks are automatically unlocked and their assigned agents are woken вҖ” no manual intervention.

### рҹӘӘ SOUL Documents
Each agent gets a structured identity document вҖ” **IDENTITY, DECISION PRINCIPLES, CYCLE CHECKLIST, PERSONALITY** вҖ” generated by LLM or hand-crafted. Solves the "Memento Problem" where agents forget who they are across sessions.

### вҸұпёҸ Configurable Heartbeat
Each agent wakes on its own schedule (5m, 15m, 1h, 24h, or custom). The heartbeat is the core of autonomy вҖ” it processes inbox tasks, runs adapters, tracks costs, and feeds results back to the team.

### рҹҸў Full Control Plane
- **Dashboard** вҖ” live agent grid, system pulse, company health score
- **War Room** вҖ” full-screen mission control with animated counters
- **Meetings** вҖ” agents can call peer meetings, synthesize results
- **Goals (OKR)** вҖ” 4-level hierarchy, progress tracking
- **Kanban** вҖ” tasks with Critic-reviewed outputs
- **Activity** вҖ” 28-day heatmap, filtered feed
- **Routines** вҖ” cron-based automations
- **Memory** вҖ” browsable agent memory with Knowledge Graph viz
- **Telegram** вҖ” mobile interface with `/status`, `/tasks`, free-form chat
- **OpenClaw Gateway** вҖ” bridge existing OpenClaw agents into your company with their full knowledge base

## Quick Start

### Option A вҖ” One-line installer (recommended)

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/OpenCognit/opencognit/main/install.sh)
```

Interactive wizard: asks for project name + optional API key, clones the repo, installs deps, generates secrets, migrates DB, builds frontend. Done in under 2 minutes.

```bash
opencognit
# вҶ’ http://localhost:3201
```

---

### Option B вҖ” Docker

```bash
git clone https://github.com/OpenCognit/opencognit.git
cd opencognit
docker compose up
# вҶ’ http://localhost:3201
```

> **Data is persisted** in `./data/` вҖ” safe across restarts.

---

### Option C вҖ” Manual Setup

**Requirements:** Node.js вүҘ 20

```bash
git clone https://github.com/OpenCognit/opencognit.git
cd opencognit
bash setup.sh   # installs deps, generates keys, migrates DB, builds frontend
npm start
# вҶ’ http://localhost:3201
```

---

### First Steps After Setup

1. **Create your company** вҖ” name it, describe its goal
2. **Configure API keys** вҖ” Settings вҶ’ add Anthropic or OpenRouter key
3. **Set default model** вҖ” Settings вҶ’ Standard-Modell (all agents use this unless overridden)
4. **Add a CEO agent** вҖ” set connection type to `ceo`, enable Orchestrator flag
5. **Add worker agents** вҖ” give them roles, skills, heartbeat intervals
6. **Create your first goal** вҖ” CEO will start breaking it down autonomously

## Self-Hosting

OpenCognit is **100% self-hosted**. Your data never leaves your machine:
- Database: SQLite file in `./data/`
- API keys: encrypted at rest
- No telemetry, no analytics, no external dependencies

The only outbound connections are the LLM API calls you configure.
